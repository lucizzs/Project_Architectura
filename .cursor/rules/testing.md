# Testing Strategy

Мінімум 200 модульних тестів. Покриття >= 70%.
Кожен метод: happy path + edge case + error case.
Репозиторії тестуються напряму (без моків).
Сервіси тестуються і з моками (ізольовано) і з реальними репо (інтеграційно).
Генерація звітів: lcov.info (SonarCloud), junit.xml (CI artifacts).
